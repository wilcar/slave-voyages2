Chapter 2 Improtation et inventaire des données

2.1 Lecture des données

Vous devez disposer des fichiers tastdb-spss-import.Ret tastdb-exp-2019.savdans votre répertoire de travail. La lecture du fichier tastdb-exp-2019.sav s’opère à l’aide du script tastdb-spss-import.Rgrace à la fonction source qui permet d’exécuter un script R à l’intérieur d’un autre script. Le script vise à substituer Les étiquettes de valeur aux valeurs du fichier spss, lire les dates et convertir les noms de variables en minuscules pour plus de cohérence avec les conventions de R.

source("tastdb-spss-import.R")

2.2 Packages

library(tidyverse)

2.3 Inspection visuelle des données

La visualisation du tableau de données dans la console à l’aide la fonction printn’est guère envisageable en raison de ses dimensions. Utilisez plutôt l’affichage sous la forme de table html (fonction View ou raccourcis dans le panneau Global environnement).

2.4 Structure des données

2.4.1 Voyages, un objet de type Tibble

  • Un tibble est une réinvention moderne du data.frame de R. C’est une matrice composée :
    • d’une marge d’en-têtes (variables names)
    • de lignes (rows, obs.)
    • de colonnes (columns, variables).
  • Chaque colonne comporte des données d’un même type.
  • Les colonnes sont nécessairement de longueur identique.
  • La notion de dimensions correspond au nombre de lignes et de colonnes composant le tableau de données.
dim(voyages)
## [1] 36002   274
